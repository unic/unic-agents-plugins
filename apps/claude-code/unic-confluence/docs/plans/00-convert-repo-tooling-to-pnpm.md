## Context

The repo currently uses npm with a `package-lock.json`. The organisation standard is pnpm, which the README already mentions for installation (`pnpm install`). Having `package-lock.json` alongside pnpm usage creates confusion and means two lockfile formats can silently diverge. Moving to pnpm workspace mode with a catalog reference for `marked` gives supply-chain hardening (pinned version in one place, `save-exact`, engine checks) and aligns the developer workflow with what the README already describes.

## Current behaviour

- `package-lock.json` exists at repo root, managed by npm.
- `package.json` (root) has no `packageManager` field and no `engines` field.
- `marked` is declared as `"marked": "17.0.5"` in `dependencies` (not a catalog reference).
- `commands/unic-confluence.md`, step 0, runs `npm install --prefix "${CLAUDE_PLUGIN_ROOT}"`.
- `README.md` documents `pnpm install` for repo setup but does not mention `pnpm confluence` for running the command (uses `npm run confluence --`).

Relevant file paths and content:

`package.json` (full current content):
```json
{

  "license": "LGPL-3.0-or-later",
```

`commands/unic-confluence.md`, step 0 (lines 10-11):

## Target behaviour

- `package-lock.json` is deleted.
- `pnpm-workspace.yaml` exists at repo root with catalog entry for `marked`.
- `.npmrc` enforces `engine-strict=true` and `save-exact=true`.
- `package.json` declares `packageManager` (exact pnpm version), `engines` (`node>=24`, `pnpm>=10`), and references `marked` via `catalog:`.
- `pnpm-lock.yaml` is generated by running `pnpm install`.
- `commands/unic-confluence.md` step 0 uses `pnpm install`.
- `README.md` uses `pnpm confluence` (not `npm run confluence --`) for all run examples and has no remaining `npm install` references in install sections.
- `node scripts/push-to-confluence.mjs` (no args) exits 1 with a usage message — not a `require()` crash — confirming `marked` still loads correctly after the lockfile swap.

## Implementation steps

### Step 1 — Delete `package-lock.json`

```sh

```

### Step 2 — Create `pnpm-workspace.yaml`

Create the file at repo root with this exact content (spaces for YAML, not tabs):

```yaml
packages:
  - "."

catalog:
  marked: "17.0.5"

catalogMode: prefer
preferWorkspacePackages: true
strictDepBuilds: true
minimumReleaseAge: 1440
trustPolicy: no-downgrade
useNodeVersion: "24.x.y"   # Ralph: replace with latest Node 24.x LTS patch from https://nodejs.org/en/download
```

### Step 3 — Create `.npmrc`

Create the file at repo root:


### Step 4 — Update `package.json`

Replace the full content of `package.json` with:

```json

	"name": "unic-confluence",
	"private": true,
	"license": "LGPL-3.0-or-later",
```
